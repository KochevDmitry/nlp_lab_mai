# Лабораторная работа 2: NLP

## Часть 1: LLM-сервис

### Архитектура

Docker-контейнер на базе `python:3.12-bookworm` содержит:
- **Ollama-сервер** с моделью Qwen2.5:0.5B
- **FastAPI-сервис**, оборачивающий Ollama одним эндпоинтом

При старте контейнера `entrypoint.sh` последовательно:
1. Запускает Ollama в фоне
2. Ожидает готовности сервера
3. Скачивает модель `qwen2.5:0.5b`
4. Запускает FastAPI на порту 8000

### Установка и запуск


**1. Собрать и запустить контейнер**
```bash
docker compose up --build
```

**2. Проверить работоспособность**
```bash
curl http://localhost:8000/health
```

Ожидаемый ответ:
```json
{"status": "ok", "ollama": "running"}
```

**3. Отправить тестовый запрос**
```bash
curl -X POST http://localhost:8000/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Что такое машинное обучение?"}'
```

**4. Запустить клиентский скрипт (вне контейнера)**
```bash
python client.py
```

Скрипт отправляет 10 запросов и сохраняет `report.json`.

### API

#### `GET /health`
Проверка состояния сервиса.

#### `POST /generate`
Отправка запроса к LLM.

Тело запроса:
```json
{
  "prompt": "Вопрос",
  "model": "qwen2.5:0.5b"
}
```

Ответ:
```json
{
  "prompt": "Вопрос",
  "response": "Ответ модели",
  "model": "qwen2.5:0.5b"
}
```

## Часть 2: Исследование техник промптинга

### Датасет

[SMS Spam Collection Dataset](https://www.kaggle.com/datasets/uciml/sms-spam-collection-dataset) — 5574 размеченных SMS-сообщения (ham/spam).

Для оценки использовалась стратифицированная выборка 100 сэмплов (80 ham / 20 spam), сохраняющая исходное соотношение классов.

### Техники промптинга

Исследовались четыре техники. Каждая требует от модели вернуть JSON:
```json
{"reasoning": "<объяснение>", "verdict": 0}
```
где `verdict = 1` — спам, `verdict = 0` — не спам.

- **Zero-shot** — только инструкция без примеров и пошагового рассуждения
- **CoT** (Chain-of-Thought) — инструкция с явным указанием думать по шагам
- **Few-shot** — инструкция с четырьмя размеченными примерами
- **CoT + Few-shot** — комбинация примеров и пошагового рассуждения

### Запуск исследования

```bash
pip install requests pandas scikit-learn
python research.py
```

Результаты сохраняются в `research_report.json`.

### Результаты

| Техника       | Accuracy | Precision | Recall | F1     |
|---------------|----------|-----------|--------|--------|
| zero_shot     | 0.6500   | 0.2727    | 0.4500 | 0.3396 |
| cot           | 0.7600   | 0.3571    | 0.2500 | 0.2941 |
| few_shot      | **0.7200**   | **0.4048**    | **0.8500** | **0.5484** |
| cot_few_shot  | 0.7100   | 0.3448    | 0.5000 | 0.4082 |

### Выводы

**Few-shot** показал лучший результат по F1 (0.548) и Recall (0.85). Высокий Recall особенно важен для задачи спам-детекции: пропустить спам хуже, чем ложно заблокировать легитимное сообщение.

**CoT** дал лучший Accuracy (0.76), но низкий Recall (0.25) — модель слишком осторожна и часто классифицирует спам как ham, что неприемлемо для данной задачи.

**CoT + Few-shot** показал средний результат и не превзошёл чистый few-shot. Вероятно, модель размером 0.5B параметров недостаточно велика, чтобы эффективно использовать длинные сложные промпты.

**Zero-shot** ожидаемо показал худший результат по всем метрикам.

**Общий вывод:** Qwen2.5:0.5B демонстрирует умеренное качество на задаче детекции SMS-спама. Few-shot промптинг наиболее эффективен для небольших моделей: конкретные примеры компенсируют ограниченные способности модели к самостоятельному рассуждению. Для продакшн-решения рекомендуется использовать модель большего размера или дообучить модель на размеченных данных.
